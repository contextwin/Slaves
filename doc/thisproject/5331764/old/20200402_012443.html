Slaves 技術サイド進捗状況

2020年01月29日

教訓：個人の考察に、代表としての責任を問うべきか考えよう。

課題
・リソース共有用のwebストレージがほしい　未
    githubリポジトリ作成　完了
    https://github.com/contextwin
    (ただしプログラミング班以外にとって操作が難しい可能性があるので別の共有方法を検討)

・png画像の表示を可能にする　可
    デコード処理準備　完了
    Xwindowで表示 完了?　(SDL2依存では表示完了)
　方向キー入力対応 完了

　png画像表示は可能になったが、描画処理の実現を急いだため、sdl2-dev,sdl2-imageに依存する状態になっている。
    描画のミッションはクリアしたが、デコードされたデータをXlibの低水準な関数で描画できる形に変換する機能を自前で実装したほうが将来的に良さそうであると考え、個人的な課題にする。
　pngのデコードでしようされた<png.h>は私の環境では既に開発環境としてインクルードできる状態であったが、場合によってはこのデコード処理も自前にしてしまったほうが良いかも知れない。
すでにsdl2-imageの内部でlibpngがリンクされている模様 <- 記事を見ただけで確認はしてない

・枠のデザイン、使用するフォントの決定　未
・メニュー画面、セレクト画面デザイン　未
・テストプレイ要員が欲しい 未

現状
枠のデザインとpng画像表示の実装から始める。グラフィック担当の用意した画風を見てから枠のデザインを始めてもいいかも知れないと考えるので、先にpng画像表示を実装することにする。
・ラフ画ではあるがpng画像処理、表示機能はlibsdl2依存で実装完了

メニュー画面とセレクト画面のデザインはグラフィック担当とアイデア出しを行ってからすすめたい。
リソース共有用のwebストレージに関しては、githubでの共有ではおそらく使用したことのないメンバーにとって操作が煩雑なのではと思われそのための学習コストも発生しそうなので避けたい。how to ドキュメントの作成なども手間である。現在AWSが良いかも知れないと考えている。
一方で個人の(contextwin）githubアカウントの方でリポジトリを作成し私が管理する。
https://github.com/contextwin

SD3等身、4等身の移動キャラグラデータ検討し、コミカルさとカッコイイの中間くらいのやつを採用。

ディレクトリ内容について
./image ... 使用する画像データ類を格納
./fonts ... 使用するフォントデータ類を格納
./doc ... 参考資料類、参考にしたリンク類の格納
./tool ... 作業効率化のためのsh,perlなどの格納

ヘッダファイルについて
./function.h ... 比較的機能層的処理でなおかつ、汎用性がありそうなマクロと関数の定義
./view.h ... 比較的抽象層的処理でなおかつ、汎用性がありそうなマクロと関数の定義
./specific.h ... 今回のプロジェクト以外ではおそらく使わないであろうマクロと関数の定義
./mysdl2.h... SDL2を利用したマクロと自己定義の関数の定義。

展望
4まで進んだらバトル画面のアイデアを出す
分割可能なpixelサイズの最初値算出コマンド作成 (perl)
亡びの風広報用、定時ツイート自動化ツール作成
亡びの風用githubアカウント作成&githubioの作成
